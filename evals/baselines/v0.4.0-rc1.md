## Targeted Post-Regression Validation

The full regression exposed two qualitative issues: an invented upper date boundary in the clean lower-bound case and an incorrect description of Oracle DATE as potentially date-only.

The prompt and expectations were corrected, then validated through targeted runs.

## Oracle Deep Review Coverage

The suite now covers:

- clean Oracle aggregation;
- conditional `NOT IN` NULL behavior;
- demonstrable `NOT IN` with an explicit NULL;
- invalid aggregation;
- `ROWNUM` before `ORDER BY` in Top-N queries;
- Oracle empty strings treated as NULL;
- `TO_DATE` without a format model and `NLS_DATE_FORMAT` dependency;
- equality against an Oracle DATE literal for a whole-day requirement;
- inclusive `BETWEEN` upper boundaries;
- clean CTE and `ROW_NUMBER` analytic-query behavior.

## Qualitative Decision

`v0.4.0-rc1` is **accepted as the technical release candidate**.

Reasons:

1. All 14 automated score gates passed in the complete regression.
2. Clean SQL remained distinguishable from conditional risks and direct correctness defects.
3. The clean analytic CTE returned no speculative performance findings or suggestions.
4. Oracle date and NULL semantics were validated through targeted cases.
5. Cross-dialect architecture remained intact.
6. Token usage and estimated cost are measured per review and per run.
7. Post-regression prompt changes were covered by focused regression tests.

## Known Limitations

- Average latency exceeded the historical 20-second target.
- Individual requests varied from approximately 14 to 36 seconds in the full regression.
- Secondary wording and scores remain probabilistic.
- Qualitative review is still required in addition to score gates.
- Schema, indexes, statistics, execution plans, and runtime data remain unavailable.
- PostgreSQL, SQL Server, and MySQL have shallower coverage than Oracle.
- PL/SQL procedural analysis is not included.

## Release Recommendation

Proceed with documentation, production deployment, and a private Oracle-focused Beta. Do not add more evaluation cases before collecting user feedback unless a release-blocking defect is discovered.
