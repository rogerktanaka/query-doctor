# Product Requirements Document (PRD)

## Project

**Query Doctor**

Version: 0.1.0 (MVP)

---

# Problem

Developers often write SQL queries without receiving structured feedback on performance, readability, maintainability or best practices.

Code reviews usually happen late in the development process and depend on experienced teammates.

There is an opportunity to provide instant AI-powered SQL reviews before the code reaches production.

---

# Target Audience

Primary audience:

- Software Developers
- Database Developers
- Oracle PL/SQL Developers
- SQL Developers
- Database Administrators (DBAs)

Future audiences:

- PostgreSQL Developers
- SQL Server Developers
- MySQL Developers
- Data Engineers

---

# Goal

Allow a developer to paste a SQL query and receive an AI-generated code review in less than 10 seconds.

---

# MVP Scope

The MVP must support:

- Paste SQL into a text area
- Submit the query
- Receive an AI review

The review should include:

- Summary
- Readability
- Performance observations
- Maintainability observations
- Best practices
- Suggested improvements

---

# Out of Scope

The MVP will NOT include:

- Authentication
- User accounts
- Saved history
- Database connections
- Execution plans
- SQL execution
- File upload
- Billing
- Team workspaces

---

# Success Criteria

The MVP is successful if a developer can:

1. Open the application.
2. Paste a SQL query.
3. Click "Review".
4. Receive a useful review generated by AI.

Nothing else is required.

---

# Non-Functional Requirements

- Response time under 10 seconds
- Responsive interface
- Simple user experience
- Secure API key handling
- Cloud deployment ready

---
